# Simple Tag List Editor

Полностью клиентское (SPA) веб-приложение для удобного составления и валидации списка тегов. Требует только статичного размещения на сервере.

[Демонстрация работы](https://abubaca4.github.io/Simple-tag-list-editor/)

[HTML to JSON Converter](https://abubaca4.github.io/Simple-tag-list-editor/html_to_json_str.html)

## Функциональность
* **Двусторонняя синхронизация:** Выбор тегов кликом и автоматическое распознавание тегов при вставке готового текста.
* **Умные зависимости:** Автоматическая активация связанных тегов при выборе основного.
* **Альтернативный вывод:** Генерация параллельного списка (например, на английском) с отдельным разделителем и опцией удаления дубликатов.
* **Валидация:** Учет лимита символов, подсветка нераспознанных слов, контроль обязательного выбора в категориях, динамическая система проверок на основе пользовательских выделений.
* **Интерфейс:** Темная/светлая темы, переключатель режима показа тегов (текст/картинка), закрепление шапки, сворачиваемая справка и индикаторы порядка выбора.
* **Автосохранение:** Приложение запоминает введенный текст и восстанавливает его при перезагрузке страницы (индивидуально для каждого конфига).

## Установка и запуск
1.  Скачайте файлы.
2.  Отредактируйте `tags.json` под свои нужды.
3.  Разместите файлы в статичном каталоге веб сервера.
4.  Откройте страницу.

Для загрузки другого конфига используйте URL-параметр: `index.html?conf=my_config` (загрузит `my_config.json`).

## Конфигурация (tags.json)

### Глобальные настройки
* `separator`: Разделитель основных тегов (например, `, `).
* `alternativeSeparator`: Разделитель альтернативных тегов.
* `alternativeName`: Название для поля альтернативных тегов (отображается над полем, если указано).
* `characterLimit`: Лимит символов (визуальное предупреждение).
* `reference`: HTML-код для блока справки.
* `webLinks`: Массив объектов для ссылок в шапке:
    * `name`: Текст ссылки.
    * `url`: Адрес.
    * `fName` (опционально): Идентификатор для фильтрации. Ссылка отобразится только если в URL страницы передан параметр `?linkbutton=идентификатор`.
    * `target`: Атрибут target для ссылки (например, "_self" для открытия в текущей вкладке, по умолчанию "_blank"). 
* `highlightedTags`: Массив конфигураций для динамических выделений тегов. Первое выделение в массиве имеет наивысший приоритет.
    * `name`: Название выделения.
    * `light`: Объект с цветами для светлой темы (`bg`, `border`, `text`).
    * `dark`: Объект с цветами для темной темы (`bg`, `border`, `text`).
* `hideInputSection`: Скрыть вывод основных тегов (если нужен только альтернативный).
* `cacheMaxAgeHours`: Через сколько часов принудительно обновить JSON из сети, игнорируя кеш браузера.
* `unrecognizedIgnore`: Массив. Для этих тегов не будет выводится предупреждение о том что они не были распознаны.
* `imageMode`: контролирует начальный режим отображения кнопок тегов. Допустимые значения:
    * `textFirst` — текстовые кнопки (по умолчанию);
    * `imageFirst` — сначала показываются картинки;
    * `imageOnly` — всегда только картинки (переключатель в шапке скрыт).

### Категории (`categories`)
Типы категорий (`type`):
1.  **standard**: Теги добавляются в фиксированном порядке (как в JSON).
2.  **single**: Радио-кнопки (можно выбрать только один тег в категории).
3.  **ordered**: Теги добавляются в порядке клика пользователя (удобно для задания приоритетов).

Параметры категории:
* `name`: Название категории.
* `requirement`: Может быть строкой или массивом строк.
    * `"atLeastOne"` (обязателен выбор любого тега). Учитывается, только если это единственное значение.
    * `["ИмяВыделения1", "ИмяВыделения2"]` (динамическая валидация). Если в категории присутствуют теги с указанным выделением (хайлайтом), то система потребует выбрать хотя бы один тег с таким выделением. Текст ошибки формируется автоматически с сохранением цветовой стилистики.
* `overrideRequirementText`: Текст ошибки валидации (переопределяет автоматический текст для `requirement`).
* `description`: Текст всплывающей подсказки рядом с названием.
* `reference`: HTML-код для блока справки для категории.

### Теги (`tags`)
* `name`: Строка (имя) или Массив строк (варианты написания). Если указан массив, первый элемент считается основным.
* `alternative`: Текст для поля альтернативного вывода.
* `highlight`: Строка или массив строк. Указывает имена выделений (из массива `highlightedTags`), которые применяются к тегу. Визуальный стиль применяется от выделения с наивысшим приоритетом, однако выбор тега закрывает требования (requirement) для всех его выделений.
* `knownAs`: Массив синонимов для поиска при вставке текста.
* `subgroup`: Имя подгруппы для визуального разделения. Если начинается с `!`, заголовок скрывается.
* `description`: Текст всплывающей подсказки.
* `image`: Ссылка на картинку. При клике по изображению выбирается первый вариант имени. Если картинки нет, элемент отображения графики не создаётся.
* **`requiredTag`**: Указывает, какой тег должен быть выбран автоматически при активации данного тега.
    * **Формат 1 (Строка):** `"requiredTag": "ИмяТега"` — ищет тег в этой же категории.
    * **Формат 2 (Объект):** `"requiredTag": { "category": "Категория", "name": "ИмяТега" }` — позволяет указать тег из любой другой категории.
    * **Формат 3 (Массив):** Позволяет указать несколько зависимостей в любом из вышеуказанных форматов. Пример: `["Тег1", { "category": "Другая", "name": "Тег2" }]`.
