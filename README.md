# Simple-tag-list-editor
Простое веб приложение, позволяющее в графическом интерфейсе составлять строку из тегов.

Под конкретные теги настраивается путём редиктирования tags.json (я предполагаю что его структура достаточно проста чтобы не требовать дополнительного описания). Доступен выбор файла конфигурации для загрузки вместо tags.json через ?conf=. Можно указывать как ?conf=1 так и ?conf=1.json в обоих случаях будет загружен 1.json.

Демонстрационный вариант примера работы https://abubaca4.github.io/Simple-tag-list-editor/

Теги разделяются разделителем, хранящимся в separator. Разделитель не должен быть пустым и не должен содержаться ни в одном из тегов.

Также есть альтернативные теги, эти теги могут быть любыми(например тот же тег картинкой или на другом языке). Разделитель для них может быть любым(хоть пустой строкой тогда теги будут идти последовательно без проблелов).

Если ни один из тегов не имеет альернативной формы тогда поле для вывода альтернативных тегов скрывается.

Есть отключаемый лимит символов. Предполагается что он используется для составления списка тегов там где максимальная длина ограничена(например в названии темы), а там, где длина не ограничена, его можно отключить для внесения более длинного списка тегов.

Теги объединяются в категории. Категории бывают 3х типов:
1. standard, все выбранные теги вносятся в результирующую строку в том порядке как они указанны в tags.json.
2. single, можно выбрать только 1 тег из категории.
3. ordered, теги выводятся в том порядке, в каком их выбрал пользователь.

В категории можно задать
1. requirement требование наличия любого тега (atLeastOne) или наличия главного тега (atLeastOneMain)
2. overrideRequirementText замена текста предупреждения
3. description описание категории

Для каждого тега можно указать 5 параметров(если параметр пуст его можно опустить):
1. name, основное имя тега, обязательно к заполнению. Может быть массивом, тогда тег имеет несколько вариантов наименования, но выбран может быть только один.
2. alternative, текст для альтернативного списка.
3. subgroup, название подгруппы, позволяет внутри категории объединять теги в более малые подгруппы. Группировка число визуальная и не влияет на логику работы категории. Если название подгруппы начинается с ! то оно скрывается.
4. description, описание которое выводится при наведении курсора мыши на тег.
5. main выделяет тег среди остальных
6. knownAs старые наименования тега по которым он будет искаться во входной строке

Теги выводятся в том же порядке, в каком расположены категории.

Если вставить строку с тегами в поле вывода, то приложение автоматически выберет те теги, которые есть в этой строке. Любые теги которых нет в tags.json будут удалены.

Приложение полностью работает на стороне клиента, никакое взаимодействие с сервером, помимо получения статичных файлов, не требуется.
